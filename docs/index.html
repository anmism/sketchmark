Why Sketchmark

Everything a diagram tool should be.
Nothing it shouldn't.

✏

Hand-drawn rendering

Every shape is rendered by rough.js — slightly wobbly, slightly imperfect. Diagrams that look drawn, not printed. Perfect for communicating ideas that are still forming.

▶

Built-in animation

Step through any diagram with step highlight, step draw, step move, step fade and more. Animate nodes, edges, groups, tables, notes, and charts — all from DSL.

⌨

Plain text DSL

No XML. No JSON. No visual editor. Write diagrams the same way you write code — in your editor, in version control, in a PR description. Diffs that actually make sense.

⚡

Layout engine

Flexbox-style layout: row, column, grid. Nested groups, justify, align, gap, padding. Auto-sizes everything. Override with explicit width/height when you need pixel control.

◑

9 theme palettes

light, dark, ocean, forest, sunset, slate, rose, midnight, sketch. One line to switch. Define custom named themes inside any diagram and apply them to any element.

↓

Export anywhere

SVG, PNG, HTML (self-contained). Transparent background support. The SVG is clean and readable. Embed in docs, presentations, GitHub READMEs, or Notion.

Animation

Step through your architecture live

Add step instructions at the bottom of any diagram. Click next to advance — nodes draw themselves, edges animate in, groups reveal, elements highlight, fade, move, and scale. Walk an audience through a system design one connection at a time.

animation.sketchmark
step highlight client
step draw    client-->gateway
step move    server dx=0 dy=-60 duration=500
step scale   gateway factor=1.3
step fade    oldbox
step color   alert fill="#ff4444"
step rotate  broken deg=25
Layout

Flexbox-style groups, nested infinitely

Groups are flex containers. Set layout=row, layout=column, or layout=grid. Nest groups inside groups. Justify content with space-between or center. The engine measures and places everything — you just describe structure.

layout.sketchmark
group services layout=column gap=16
{
  group tier1 layout=row justify=space-between
  {
    box auth  label="Auth"
    box api   label="API"
    box cache label="Cache"
  }
  group dbs layout=grid columns=2
  {
    cylinder pg    label="Postgres"
    cylinder redis label="Redis"
  }
}
Themes

One line. Nine palettes. Infinite custom.

Switch the entire diagram with config theme=ocean. Or define named themes inline — reuse them across every element. Sketchy pencil-on-paper, deep ocean blues, midnight code-editor dark. Your diagram, your palette.

themes.sketchmark
# built-in palette
config theme=sketch

# or define your own
theme brand fill="#f0e8ff" stroke="#7700cc"
theme warn  fill="#fff3cd" stroke="#f0a500"

box a theme=brand
box b theme=warn

Who it's for

Built for people who think
in systems.

Software Engineers

Architecture docs that live in git

Write diagrams alongside code. Review them in PRs. No binary files, no drag-drop sessions, no "who has the Lucidchart link?" — just text, committed, versioned, owned.

Engineering Managers

System walkthroughs that actually walk

Use step animations to guide teams through a new architecture. Reveal each service as you explain it. Highlight the bottleneck. Fade out the old system. Present, don't just show.

Educators & Tech Writers

Teach concepts with progressive reveal

Explain TCP handshakes, OSI layers, auth flows, microservice patterns. Build diagrams that unfold step by step. The hand-drawn style signals "this is an explanation, not a spec."

Developers building tools

Embed diagrams in your own product

Sketchmark is an npm package. Call render() with a DSL string. Get back an SVG, an animation controller, export functions. Integrate into docs sites, wikis, notebooks, dashboards.

Get started

Two lines to your first
diagram.

$ npm install sketchmark roughjs
your-first-diagram.js
import { render } from 'sketchmark';

render({
  container: document.getElementById('diagram'),
  dsl: `diagram
  box a label="Hello"
  box b label="World"
  a --> b label="connects"
  step draw a
  step draw a-->b
  step draw b
  end`,
});

rough.js is a peer dependency — load it before sketchmark.
Works in any bundler (Vite, webpack, Next.js) or directly via CDN with an import map.
